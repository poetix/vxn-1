## Summary

Surface the E004 additions on the editor's per-layer faceplate: a second LFO
section (shape / rate / delay), the new sixth matrix row (Lfo2 → pitch / cutoff
/ amp / PWM), and the per-LFO sync + reset toggles for both LFOs. Rides the
existing Upper/Lower layer-toggle UI from E003 (ticket 0013).

> **Reinterpreted after [E005](../../epics/closed/E005-per-voice-and-global-lfo.md).**
> The LFOs are no longer a symmetric per-layer pair: LFO 1 is per-voice (shape /
> rate / sync **plus** a delay→fade onset and free-run toggle), and LFO 2 is one
> global instrument-wide oscillator (global shape / rate / sync; per-patch
> routing depths). The "reset" toggle was superseded (a per-voice LFO retriggers
> intrinsically), so the LFO 1 panel surfaces delay/fade/free-run in its place.

## Acceptance criteria

- [x] LFO 2 controls (shape selector, rate, sync) on a dedicated **global**
      panel, bound to the `Lfo2*` global params. LFO 1 gains its per-voice
      delay / fade / free-run controls (the E005 replacement for "reset").
- [x] The Lfo2 matrix row's four per-patch depths (`Lfo2Pitch/Cutoff/Amp/Pwm`)
      are surfaced as faders beside their LFO 1 counterparts (Vib2 / PWM2 in VCO
      Mod, LFO2 in Filter, Trem2 in Amp) — matching the existing contextual
      matrix layout rather than a generic 6×4 grid.
- [x] Per-LFO sync toggle (`LfoSync` per-patch / `Lfo2Sync` global). Reset toggle
      dropped per E005; LFO 1 free-run + delay/fade surfaced instead.
- [x] When an LFO's sync is on, its rate readout shows the musical subdivision
      (e.g. `1/8`) instead of Hz; off shows Hz. (`sync_partner` + the shared
      `SUBDIVISIONS` table.)
- [x] LFO 1 panel + the `Lfo2*` routing depths follow the Upper/Lower edit
      target (per-patch); LFO 2 shape/rate/sync stay fixed (global). Whole mode
      handled as for the rest of the faceplate.
